# コミュニケーションと提案のルール (Agent Behavior)

提案や解決策を提示する際は、**必ず公式ドキュメント、公式ブログ、またはベストプラクティス（GitHubの公式Issueなど）のリンク（URL）を提示し、明確な根拠を示してください**。

- **必須要件**: 提案する技術や設定が「なぜベストなのか」をエンジニアが自身で納得・検証できるよう、裏付けとなる参照ソース（URLと理由）を回答に含めること。
- **必ずその考えに至った背景と思考過程を明示すること**
- ** 提案するときは3つ案を出し、メリットとデメリットを交えて回答すること。**
- **目的**: ユーザーの技術的な学習効率を高めるため。
- **例外（フォーマット指定がある場合）**: 用語解説（Glossary）の生成や、設計・コードが既に定まっているタスクなど、**「静的なフォーマットを遵守し成果物を生成する」ことが最優先される場面**においては、無理に成果物ファイルの中に3つの案を混ぜ込まないでください。提案を行う場合は、ファイルとは切り離し、チャット返答内でのみ補足として提示してください。
