# 用語集に追加

引数: $ARGUMENTS

`.agent/rules/glossary.md` と `.agent/skills/glossary/SKILL.md` の品質基準とフォーマットに従ってください。

## 手順

### 1. 用語の特定

$ARGUMENTS で指定された用語を特定する。

### 1.5. 重複チェック

`docs/glossary.md` に同一または類似の用語が既に存在しないか確認する。
既に存在する場合は、更新が必要か判断し、ユーザーに報告する。

### 2. 調査

必要に応じてウェブ検索を行い、最新の公式ドキュメントや比較記事を参照する。
情報源の優先順位: 公式ドキュメント → GitHub リポジトリ → 技術ブログ → カンファレンス資料

### 3. 解説生成

以下のフォーマットで解説を生成する：

```markdown
### [用語名]

#### 何をしているか

（平易な言葉で、このパーツの役割を解説）

#### なぜ必要か

（システム全体におけるこのパーツの位置づけ。なぜ代替技術ではなくこれなのか）

#### 公式ドキュメント

- [ドキュメントタイトル](URL)

#### 類似技術との比較

| 技術 | 特徴 | 選定理由（または差別化ポイント） |
| ---- | ---- | -------------------------------- |
| ...  | ...  | ...                              |

#### 引用と解釈

**引用元**: [ドキュメント名](URL) - セクション名

> 引用した具体的な文言

**解釈**: この引用をこのプロジェクトにどう適用したか、どう解釈したかを説明。

#### 面接で聞かれそうな質問

1. **Q**: ...
   **A**: ...

#### 備考

（プロジェクト固有の注意点など。該当しなければ省略可）
```

### 4. ファイル更新

`docs/glossary.md` の末尾に追記する。

### 5. 報告

追加完了を報告し、簡易的な解説を表示する。
